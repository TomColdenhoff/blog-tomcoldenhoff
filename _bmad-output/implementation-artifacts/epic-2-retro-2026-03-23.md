## Deep story analysis — patterns

### What went well

1. **Single validation stack** — 2.2 established `published-post-frontmatter.mjs` / `validate-published-posts.mjs`; 2.3 and 2.4 extended it instead of spawning parallel pipelines. That kept CI coherent and errors consistent (`::error file=...::`).
2. **Review culture** — Multiple stories record **Senior Developer Review (AI)** with triage, fixes, and re-approval (e.g. argv/`publicDir` bug in draft verification, shell-safe git in slug stability, taxonomy integration tests and `verify-taxonomy-list-pages.mjs`).
3. **Explicit policy docs** — URL change policy, taxonomy conventions, and publishing checklist/runbook give authors a **system** to follow, aligned with architecture enforcement language.
4. **Fixtures as living proof** — `draft-fixture-not-listed.md`, `hello-world.md`, and post-build scripts make draft exclusion and taxonomy routes **continuously verifiable**, not tribal knowledge.

### Struggles and risks (systems, not blame)

1. **Validator vs real YAML** — Stories document that the simple frontmatter parser supports **inline** taxonomy arrays for CI; block-style lists are deferred. Authors must follow documented patterns or CI will disagree with Hugo.
2. **Slug-stability automation limits** — Git rename as delete+add can bypass comparison; `url:` override drift was explicitly called out and partially addressed. Policy + CI are **good but not complete** — runbook and human review still matter.
3. **Cross-cutting coupling** — Taxonomy rules flow into **slug-stability** classification (`validatePublishablePost`). Comments and docs now point to this; future relaxations need **two places** updated.
4. **Scope creep resistance** — 2.4 and 2.5 correctly **deferred** full Epic 3 navigation polish while still shipping routable taxonomy pages and a menu-linked About page.

### Recurring review themes

- **CLI safety:** optional path args must not consume flags (`--ci`).
- **Fail-closed:** invalid head frontmatter, alias requirements vs ack-only automation.
- **Docs = product:** runbook/checklist updates whenever CI behavior changes.
