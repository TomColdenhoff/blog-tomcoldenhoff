# Taxonomy conventions (categories and tags)

Published posts use Hugo’s built-in **`categories`** and **`tags`** taxonomies. This document is the source of truth for term shape and pillar rules; CI enforces it via **`scripts/validate-published-posts.mjs`** (see **`scripts/published-post-frontmatter.mjs`**).

## Term format

Every string in **`tags`** and **`categories`** must:

- Use **lowercase ASCII** letters and digits only, with **single hyphens** between word segments.
- Match: **`^[a-z0-9]+(-[a-z0-9]+)*$`** (same rule enforced in CI via **`scripts/published-post-frontmatter.mjs`**)  
  Examples: `ai`, `machine-learning`, `founder-ops`  
  Invalid: `AI`, `Machine Learning`, `snake_case`, `-edge`, `trailing-`

**YAML shape for CI:** This repo’s validator only understands **single-line** frontmatter values, including **inline bracket arrays** (e.g. `tags: [a, b]`, `categories: [software]`). Avoid block-style lists under `tags:` / `categories:` until the parser supports them—use inline `[]` so local **`node scripts/validate-published-posts.mjs`** matches what you intend.

**Draft posts** (`draft: true`) are not checked for taxonomy content; still prefer valid terms when you draft so publish is frictionless.

## Pillar categories (primary shelf)

These four slugs are the **canonical topic pillars** for the site:

| Pillar slug        | Use for |
|--------------------|--------|
| `entrepreneurship` | Building companies, GTM, founder operations |
| `ai`               | ML, agents, product AI, research notes in AI |
| `software`         | Engineering, systems, tooling, craft |
| `science`          | Research and scientific topics outside pure software |

**Published posts** (`draft: false`): **`categories`** must be a non-empty YAML array that includes **at least one** of the pillars above. Additional categories are allowed if they also satisfy the term format regex (prefer **one** pillar per post; extra categories only when clearly useful).

**Tags** are optional. Use them for finer subtopics (still hyphenated lowercase). Example: `categories: [software]` and `tags: [hugo, static-sites]`.

## URL stability

Renaming or removing a category or tag changes the URL for that term’s **taxonomy list page** and can break bookmarks. Prefer **stable** terms from the start.

**Not the same as post slug policy:** **`docs/url-change-policy.md`** governs **post** permalinks (`slug`, `url`, `aliases` on posts). Taxonomy term URLs are separate routes (e.g. `/categories/<term>/`). Mitigating a renamed term usually means keeping the old term in content until you add a replacement strategy (e.g. Hugo term **`aliases`** if you use them, or updating internal links)—do not assume post-level alias rules alone fix taxonomy URLs.

## Reader navigation

Taxonomy **list pages** are generated by Hugo (e.g. `/categories/<term>/`, `/tags/<term>/` with default pluralized paths). Global nav that surfaces categories/tags in the header is tracked under **Epic 3** (e.g. story 3.3); this story only guarantees correct metadata and routable list output.
